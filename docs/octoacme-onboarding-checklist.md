# OctoAcme — Team Onboarding Checklist

## Purpose
Provide a standardized onboarding experience for new team members joining OctoAcme projects. This checklist ensures that all team members, regardless of role, understand the project, the process, and their responsibilities.

## Pre-Onboarding (Before Day 1)

- [ ] Confirm team member's role and responsibilities
- [ ] Share project charter and one-pager
- [ ] Add to project communication channels (Slack, email lists, etc.)
- [ ] Grant access to repository, project board, and documentation
- [ ] Schedule initial 1:1 with Project Manager
- [ ] Identify buddy/peer mentor for first week

## Day 1 Orientation

### General Orientation
- [ ] Welcome meeting with Project Manager (30 min)
- [ ] Overview of OctoAcme process and governance (docs/octoacme-project-management-overview.md)
- [ ] Tour of repository structure and key documentation
- [ ] Introduce to team members and stakeholders
- [ ] Review communication channels and meeting schedule

### Role-Specific Orientation
**All Roles:**
- [ ] Review roles and personas document (docs/octoacme-roles-and-personas.md)
- [ ] Review cross-functional collaboration guide (docs/octoacme-cross-functional-collaboration.md)
- [ ] Understand escalation paths and blocker management

**Developers:**
- [ ] Code repository tour and development setup
- [ ] Review branching and PR conventions
- [ ] Run CI/CD pipeline locally
- [ ] Review acceptance criteria and Definition of Done

**Product Manager / Product Manager:**
- [ ] Review product vision and success metrics
- [ ] Understand backlog prioritization approach
- [ ] Meet with Business Analyst if assigned

**Project Manager:**
- [ ] Review project plan and timeline
- [ ] Understand risk register and dependency map
- [ ] Access to project board and reporting dashboard

**UI/UX Designer:**
- [ ] Review design system and specifications
- [ ] Introduction to Developers who will implement designs
- [ ] Access to design tools and collaboration spaces

**DevOps Engineer:**
- [ ] Review infrastructure and deployment pipeline
- [ ] Understand environment configuration and access
- [ ] Review deployment checklist and rollback procedures

**QA Professional:**
- [ ] Review test plan and acceptance criteria
- [ ] Set up test environments
- [ ] Introduction to Developers for test strategy discussion

**Business Analyst:**
- [ ] Review business requirements and user stories
- [ ] Meet with stakeholders and support team
- [ ] Understand requirement documentation standards

**Support / Customer Success Professional:**
- [ ] Review product features and known issues
- [ ] Access to support ticket system and knowledge base
- [ ] Introduction to Developers and QA for questions

**Sponsor / Executive Stakeholder:**
- [ ] Review business case and ROI expectations
- [ ] Understand decision gates and governance
- [ ] Confirm reporting cadence and metrics

## First Week

### Daily
- [ ] Attend daily standup (understand format and contribute)
- [ ] Check-in with assigned buddy/mentor
- [ ] Ask questions freely — no "dumb questions"

### Mid-Week (Day 3-4)
- [ ] Complete role-specific training or pair sessions
- [ ] Review first assigned work item or task
- [ ] Participate in project sync or planning meeting
- [ ] Provide feedback on onboarding experience

### End of Week
- [ ] 1:1 with Project Manager to discuss week and answer questions
- [ ] Share onboarding feedback and suggestions for improvement
- [ ] Confirm understanding of next week's activities

## First Month

### Week 1-2
- [ ] Complete initial task or work item
- [ ] Shadow peer in their role-specific activities
- [ ] Attend retrospective (if applicable) to see team reflection process
- [ ] Understand current risks and blockers

### Week 2-3
- [ ] Lead or co-own a small task
- [ ] Participate in peer code review (if Developer) or equivalent
- [ ] Attend stakeholder sync or customer meeting (if relevant role)
- [ ] Deepen understanding of project dependencies

### Week 3-4
- [ ] Own a complete task from start to finish
- [ ] Provide input or feedback on project direction
- [ ] Help onboard next team member or mentor peer
- [ ] 30-day check-in with Project Manager on progress and fit

## Role-Specific Learning Paths

### Developer Onboarding (First Month)
- [ ] Local development environment set up and verified
- [ ] Understand codebase architecture and key modules
- [ ] Submit first PR (with mentorship)
- [ ] Code review at least 3 peer PRs
- [ ] Understand CI/CD pipeline and how to debug failures
- [ ] Know how to find and run tests
- [ ] Understand deployment process and how to coordinate with DevOps

### Product Manager Onboarding (First Month)
- [ ] Meet with key stakeholders and customers
- [ ] Understand current backlog and prioritization rationale
- [ ] Review past sprint retrospectives and learnings
- [ ] Define or validate success metrics for your area
- [ ] Participate in backlog refinement session
- [ ] Conduct or attend user research activity

### Project Manager Onboarding (First Month)
- [ ] Review complete project plan and timeline
- [ ] Understand all identified risks and dependencies
- [ ] Review previous project boards and reports
- [ ] Facilitate a planning or standup meeting
- [ ] Prepare and deliver first status report to stakeholders
- [ ] Meet one-on-one with each team member

### UI/UX Designer Onboarding (First Month)
- [ ] Review existing design system and specifications
- [ ] Understand current user research and personas
- [ ] Review accessibility standards and compliance requirements
- [ ] Participate in design review or critique
- [ ] Create or update design documentation
- [ ] Conduct or observe usability testing session

### DevOps Engineer Onboarding (First Month)
- [ ] Complete infrastructure and pipeline walkthrough
- [ ] Understand all environments (dev, staging, prod)
- [ ] Execute a practice deployment and rollback
- [ ] Review monitoring and alerting configuration
- [ ] Participate in incident response (if occurs)
- [ ] Document lessons learned from infrastructure review

### QA Professional Onboarding (First Month)
- [ ] Understand test strategy and test plan
- [ ] Set up test environments and data
- [ ] Execute existing test cases
- [ ] Create new test cases for current work
- [ ] Participate in quality gate review
- [ ] Learn defect tracking and severity classification

### Business Analyst Onboarding (First Month)
- [ ] Meet with stakeholders to understand business context
- [ ] Review existing requirements and user stories
- [ ] Participate in requirements refinement session
- [ ] Create or refine user stories with acceptance criteria
- [ ] Document business process or workflow
- [ ] Learn documentation standards and tools

### Support / Customer Success Onboarding (First Month)
- [ ] Observe customer support interactions
- [ ] Review product features and known issues
- [ ] Learn support ticketing and escalation process
- [ ] Create or contribute to knowledge base article
- [ ] Participate in pre-release customer communication
- [ ] Understand customer feedback loop to product team

### Sponsor / Executive Onboarding (First Month)
- [ ] Understand full project business case and timeline
- [ ] Review decision gates and governance process
- [ ] Identify key risks requiring executive attention
- [ ] Confirm reporting cadence and metrics dashboard
- [ ] Meet with Project Manager and Product Manager
- [ ] Attend first major milestone review or gate

## Feedback & Continuous Improvement

### For New Team Members
- [ ] Provide feedback on onboarding experience at end of first month
- [ ] Suggest improvements to checklist or process

### For Project Manager / Onboarding Lead
- [ ] Review feedback from each new team member
- [ ] Update checklist based on learnings
- [ ] Share improvements with team

## Onboarding Sign-Off

**New Team Member:**
- [ ] I understand my role and responsibilities
- [ ] I understand OctoAcme project management process
- [ ] I know how to communicate and escalate issues
- [ ] I have completed initial assigned work or task
- [ ] I have access to all necessary tools and systems

**Project Manager / Onboarding Lead:**
- [ ] Onboarding activities completed
- [ ] New team member is productive and engaged
- [ ] Feedback collected and shared for improvements
- [ ] Buddy/mentor relationship established
